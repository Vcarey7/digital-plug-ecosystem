PRODUCT DESIGN FRAMEWORK:
```
STAGE 1 — PROBLEM DEFINITION:
Who exactly has this problem?
  → Define the persona precisely
  → Quantify how many people have it
  → Estimate the annual cost of the problem
  → Document why existing solutions fail

STAGE 2 — SOLUTION ARCHITECTURE:
What is the minimum product that solves the problem?
  → Define core functionality (what it does)
  → Define the experience (how it works)
  → Define the output (what the user gets)
  → Define what it does NOT do (scope containment)

STAGE 3 — REVENUE MODEL:
How does this make money?
  → Pricing model (one-time / subscription / usage)
  → Price point validation (what will they pay?)
  → Revenue potential (market size × conversion rate)
  → Upsell path (what does a happy client buy next?)

STAGE 4 — BUILD REQUIREMENTS:
What does this need to exist?
  → Knowledge required (what expertise is in the KB?)
  → Technology required (agents, automation, infrastructure)
  → Time to build (realistic estimate)
  → Cost to build (API costs, development time)
  → Dependencies (what must exist first)

STAGE 5 — HANDOFF PACKAGE:
What does the receiving division need to execute?
  → Complete product spec document
  → Revenue model with assumptions
  → Go-to-market brief for SIGNAL
  → Technical requirements for ATLAS/NOVA
```
