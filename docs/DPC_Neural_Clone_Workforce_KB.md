## THE ADAPTATION ENGINE

### How the Learning Loop Is Built Technically

**Step 1: Outcome Capture (n8n webhook)**
Every significant agent output triggers an outcome tracking event.
The event records: what was recommended, context, and timestamp.

**Step 2: Outcome Confirmation (30-day delayed trigger)**
30 days after the recommendation, n8n checks:
- For deals: Did HubSpot deal stage advance or close?
- For content: What were Buffer/platform analytics?
- For financial forecasts: What did the actual numbers show?
- For risk assessments: Did the predicted outcome occur?

**Step 3: Pattern Extraction (Claude API)**
Claude analyzes the recommendation + outcome pair:
```
System: You are a pattern extraction engine.
Given an agent recommendation and its outcome, extract:
1. What condition led to this outcome
2. What rule should be added to the agent's Layer 3
3. What existing rule (if any) should be updated
4. Confidence level in this new rule (based on sample size)

Only add rules with 2+ confirming examples.
Mark rules as "provisional" until 3+ examples.
Mark rules as "established" at 5+ examples.
Mark rules as "core" at 10+ examples.
```

**Step 4: KB Update (Supabase write)**
Confirmed patterns write to the agent's Layer 3 in Supabase:
```sql
INSERT INTO agent_patterns (
  agent_name,
  pattern_category,
  condition,
  outcome,
  rule_text,
  confidence_level,
  example_count,
  created_at,
  last_confirmed_at
) VALUES (...)
ON CONFLICT (agent_name, condition)
DO UPDATE SET
  example_count = agent_patterns.example_count + 1,
  confidence_level = CASE
    WHEN example_count >= 10 THEN 'core'
    WHEN example_count >= 5 THEN 'established'
    WHEN example_count >= 3 THEN 'provisional'
    ELSE 'provisional'
  END,
  last_confirmed_at = NOW();
```

**Step 5: Layer 3 Injection (next agent call)**
Before every agent API call, n8n pulls Layer 3:
```
All patterns where:
  agent_name = [this agent]
  confidence_level IN ('established', 'core')
  relevant_to = [current context category]
Order by: example_count DESC
Limit: 20 most confirmed patterns
```

And injects them into the system prompt:
```
═══ LEARNED PATTERNS (Layer 3) ═══
These are rules derived from actual outcomes in YOUR business.
They override generic best practices. They are specific to Digital Plug Co.

CORE PATTERNS (10+ examples, highest confidence):
[Rules here]

ESTABLISHED PATTERNS (5-9 examples):
[Rules here]

PROVISIONAL PATTERNS (2-4 examples, use with judgment):
[Rules here]
═══════════════════════════════════
```
